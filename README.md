# Gran Publicador

**Gran Publicador** — современная платформа для управления контентом и публикациями в социальных сетях, реализованная в формате Telegram Mini App. Система позволяет создавать проекты, управлять каналами в различных социальных сетях, планировать и автоматизировать публикации.

## ✨ Возможности

### Управление проектами
- Создание и управление множественными проектами
- Система ролей и прав доступа (Owner, Admin, Editor, Viewer)
- Приглашение участников в проекты
- Архивирование проектов с виртуальным каскадным удалением

### Управление каналами
- Поддержка множественных социальных сетей (Telegram, Instagram, VK, YouTube, TikTok, X, Facebook, LinkedIn, собственные сайты)
- Привязка каналов к проектам
- Управление учетными данными для каждого канала
- **Активация/деактивация каналов** — временная приостановка публикаций без потери видимости
- **Архивирование каналов** — долгосрочное хранение неиспользуемых каналов

#### Неактивные vs Архивированные каналы

| Аспект | Неактивный канал | Архивированный канал |
|--------|------------------|----------------------|
| **Назначение** | Временная пауза | Долгосрочное хранение |
| **Видимость** | Виден в списках с индикатором | Скрыт, доступен в разделе архива |
| **Публикация** | Заблокирована | Заблокирована |
| **Использование** | Сезонные каналы, обновление токенов, A/B тестирование | Больше не нужен, но данные сохранены |

**Подробнее:** [dev_docs/inactive_channels_concept.md](dev_docs/inactive_channels_concept.md)

### Публикации и посты
- Создание публикаций с различными типами контента (Post, Article, News, Video, Short, Story)
- **Единый язык публикации** — язык фиксируется при создании и не может быть изменён; все каналы публикации должны соответствовать выбранному языку
- Планирование публикаций с указанием даты и времени
- Автоматическая публикация по расписанию
- Статусы публикаций (Draft, Ready, Scheduled, Published, Failed, Expired)
- Связь публикаций с несколькими каналами (одного языка)
- **Сбор исходных текстов** — сохранение оригинальных материалов и ссылок (source texts) для каждой публикации
- **Копирование публикаций** — дублирование контента и медиафайлов в другой проект или личные черновики

### Управление медиафайлами
- Загрузка файлов напрямую или по ссылке
- Поддержка Telegram медиасистемы (file_id)
- Proxy-стриминг через внешний микросервис Media Storage
- Автоматическое определение типов медиа (Image, Video, Audio, Document)
- **Сбор метаданных изображений и автоматическая оптимизация** (перевод в WebP/AVIF, сжатие)
- Группировка медиа в альбомы
- Единое хранилище (Media Gallery) для повторного использования файлов
- **Интеграция с Unsplash** — поиск и добавление стоковых изображений в библиотеку контента

### AI Генерация контента (LLM)
- Интеграция с **LLM Router** для генерации текстов
- **Backend-интеграция**: Сборка промптов и управление контекстом на стороне сервера
- **Умная обработка контекста**: Автоматическое усечение контента и исходных текстов с учетом лимитов токенов
- **Шаблоны промптов**: Личные, проектные и системные шаблоны с управлением порядком и видимостью
- **Счетчик токенов**: Примерная оценка размера запроса на фронтенде
- **Голосовой ввод (STT)**: Запись аудио и автоматическая транскрипция через WebSocket
- **Система безопасности**: Проверка владения и прав доступа для всех AI-операций и шаблонов

### Перевод контента
- Интеграция с **Translation Gateway** микросервисом
- Поддержка нескольких провайдеров (anylang, Google, DeepL и др.)
- Авто-определение исходного языка

### Уведомления
- Push-уведомления в реальном времени через **WebSocket (Socket.IO)**
- REST API для управления уведомлениями (прочитано/непрочитано)
- Автоматическая очистка старых уведомлений

### Связи публикаций
- Группировка публикаций в **серии** (`SERIES`) и **переводы** (`TRANSLATION`)
- Создание связанных публикаций на основе существующих

### Архивирование и управление данными
- Мягкое удаление с возможностью восстановления
- Виртуальное каскадное архивирование связанных сущностей
- Перемещение сущностей между проектами
- Окончательное удаление архивированных элементов

### Интеграции
- **n8n Node** — официальная нода для интеграции с платформой n8n (`n8n-nodes-bozonx-gran-publicador`). Позволяет автоматизировать создание черновиков, публикаций и постов из внешних систем.
- **External App API** — REST API для интеграции внешних приложений (VFS, STT, LLM). Поддерживает Connect Flow и ручные токены. См. [docs/external-integrations.md](docs/external-integrations.md).

### Безопасность и аутентификация
- Авторизация через Telegram (проверка initData)
- JWT токены для фронтенд приложения
- API токены для внешних интеграций
- Scope-based доступ для API токенов (ограничение по проектам)
- Защита всех эндпоинтов

## 🛠 Технологический стек

### Backend
- **NestJS 11** — модульный фреймворк для Node.js
- **Fastify** — высокопроизводительный веб-сервер

- **Redis** — кеш и pub/sub (ioredis)
- **Socket.IO** — WebSocket для уведомлений и STT
- **JWT** — аутентификация и авторизация
- **Passport** — стратегии аутентификации
- **class-validator** — валидация DTO
- **Pino** — структурированное логирование

### Frontend
- **Nuxt 4** — Vue.js фреймворк с SSR поддержкой (в режиме SPA)
- **Vue 3** — прогрессивный JavaScript фреймворк
- **Nuxt UI 4** — компонентная библиотека на базе Tailwind CSS
- **Pinia** — управление состоянием
- **TipTap** — rich-text редактор
- **@tma.js/sdk-vue** — интеграция с Telegram Mini Apps
- **@nuxtjs/i18n** — интернационализация
- **Tailwind CSS 4** — utility-first CSS фреймворк

### DevOps
- **Docker** — контейнеризация
- **pnpm** — быстрый пакетный менеджер
- **Jest** — тестирование
- **ESLint** — линтинг кода
- **Prettier** — форматирование кода

## 🚀 Быстрый старт

- **Node.js** >= 22.0.0
- **pnpm** >= 10.13.1
- **Telegram Bot Token** (получить у [@BotFather](https://t.me/BotFather))

### Установка

```bash
# Клонирование репозитория
git clone <repository-url>
cd gran-publicador

# Установка зависимостей бэкенда
pnpm install

# Установка зависимостей фронтенда
cd ui && pnpm install && cd ..
```

## ⚙️ Установка и настройка

### 1. Настройка переменных окружения

Для разработки:
```bash
cp .env.development.example .env
```

Для продакшн:
```bash
cp .env.production.example .env
```

Отредактируйте `.env` файл. Основные обязательные параметры: `DATABASE_URL`, `JWT_SECRET`, `SOCIAL_POSTING_SERVICE_URL`.

### 2. Инициализация базы данных

```bash
# Создание миграций и базы данных
npx prisma migrate dev --name init

# Или только применение миграций (для продакшн)
npx prisma migrate deploy

# Генерация Prisma клиента
npx prisma generate

# Наполнение базы тестовыми данными (опционально)
npx prisma db seed
``` 

По умолчанию сид не удаляет существующие данные и работает идемпотентно. Для полной очистки данных перед сидом используйте:

```bash
SEED_RESET=true npx prisma db seed
```

### 3. Настройка Telegram Mini App

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен бота и добавьте его в `.env`
3. Настройте Mini App через [@BotFather](https://t.me/BotFather):
   - Команда: `/newapp`
   - Укажите URL вашего приложения (для разработки: `http://localhost:3000`)
   - Загрузите иконку и описание

Telegram бот (если включен через `TELEGRAM_BOT_ENABLED=true`) сохраняет сообщения из личного чата в **Content Library** как `ContentItem` с блоками (`ContentBlock`) и медиа.

В **Content Library** для вкладок типа **Group** поддерживается иерархия подгрупп (дерево):

- можно создать подгруппу прямо из активной группы и сразу перейти в неё;
- при мультивыборе на вкладке `Group` доступно действие **"В группу"**;
- в модале **"В группу"** доступны режимы:
  - **Move** — удаляет связь только с текущей активной группой и переносит в выбранную подгруппу;
  - **Link** — добавляет дополнительную связь с выбранной подгруппой без удаления существующих.

## 💻 Запуск в режиме разработки

### Подготовка окружения (Docker)

Для быстрой настройки базы данных (Postgres) и кэша (Redis) используйте подготовленный Docker Compose файл.

```bash
# Копируем пример конфигурации
cp docker-compose.example.yml docker-compose.yml

# Запускаем сервисы (Postgres, Redis)
docker compose up -d
```

### Backend

```bash
# Установка зависимостей
pnpm install

# Применение миграций
npx prisma migrate dev

# Запуск с hot-reload
pnpm start:dev
```

Сервер будет доступен на `http://localhost:8080`. API эндпоинты: `http://localhost:8080/api/v1/*`.

### Frontend

```bash
cd ui
pnpm dev
```

Интерфейс будет доступен на `http://localhost:3000`.

## 🚢 Продакшн развертывание

Подробное руководство: [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md).

Основные этапы:
1. **Применение миграций БД** — используйте отдельный Render Job (`pnpm db:migrate:deploy`) или `render.yaml` Blueprint.
2. **Запуск бэкенда (API)** — запуск основного сервера (HTTP/WebSocket) через Docker image из GHCR (`node dist/src/main.js`).
3. **Запуск фонового воркера (Worker)** — отдельный контейнер из того же образа для обработки очередей публикаций (`node dist/src/worker.js`).
4. **UI деплоится отдельно** — настраивается через `NUXT_PUBLIC_API_BASE`.

### Миграции на Render.com

Для безопасного применения миграций используйте один из методов:
- **Render Job** (рекомендуется) — отдельная задача для миграций
- **Render Blueprint** — автоматизация через `render.yaml`
- **Manual Shell** — разовое выполнение через Dashboard

Подробности в [docs/DEPLOYMENT.md](docs/DEPLOYMENT.md#database-migrations-on-rendercom).


## 🔧 Конфигурация

### Переменные окружения

| Переменная | Обязательная | Описание |
|------------|--------------|----------|
| `DATABASE_URL` | Да | Строка подключения к PostgreSQL |
| `JWT_SECRET` | Да | Секрет для JWT токенов и AES-256 (минимум 32 символа) |
| `TELEGRAM_BOT_TOKEN` | Да | Токен Telegram бота от @BotFather |
| `SOCIAL_POSTING_SERVICE_URL` | Да | URL микросервиса постинга в соцсети |
| `MEDIA_STORAGE_SERVICE_URL` | Да | URL микросервиса хранения медиафайлов |
| `FREE_LLM_ROUTER_URL` | Да | URL микросервиса LLM Router |
| `REDIS_URL` | Нет | URL подключения к Redis, поддерживает Upstash (по умолчанию `redis://localhost:6379/0`) |
| `STT_SERVICE_URL` | Нет | URL микросервиса Speech-To-Text |
| `TRANSLATE_SERVICE_URL` | Нет | URL микросервиса перевода |
| `NEWS_SERVICE_URL` | Нет | URL микросервиса новостей |
| `TELEGRAM_BOT_ENABLED` | Нет | Включить Telegram бота (по умолчанию `false`) |
| `TELEGRAM_ADMIN_ID` | Нет | Telegram ID супер-администратора |
| `TELEGRAM_MINI_APP_URL` | Нет | Base URL для Telegram Mini App |
| `FRONTEND_URL` | Нет | URL фронтенда (для ссылок в сообщениях бота) |
| `LISTEN_HOST` | Нет | Хост сервера (по умолчанию `0.0.0.0`) |
| `LISTEN_PORT` | Нет | Порт сервера (по умолчанию `8080`) |
| `LOG_LEVEL` | Нет | Уровень логирования (`debug`, `info`, `warn`, `error`) |
| `SCHEDULER_WINDOW_MINUTES` | Нет | Окно обработки просроченных/запланированных публикаций (мин) |
| `SYSTEM_API_SECRET` | Нет | Секрет для системных эндпоинтов |
| `SYSTEM_API_IP_RESTRICTION_ENABLED` | Нет | Ограничить System API локальной сетью (по умолчанию `true`) |
| `SHUTDOWN_TIMEOUT_SECONDS` | Нет | Таймаут graceful shutdown (сек) |
| `HTTP_RETRY_MAX_ATTEMPTS` | Нет | Кол-во попыток для исходящих HTTP запросов (по умолчанию `3`) |
| `HTTP_RETRY_INITIAL_DELAY_MS` | Нет | Начальная задержка backoff (мс, по умолчанию `500`) |
| `HTTP_RETRY_MAX_DELAY_MS` | Нет | Максимальная задержка backoff (мс, по умолчанию `5000`) |
| `TZ` | Нет | Часовой пояс приложения (по умолчанию `UTC`) |
| `BULL_CONCURRENCY` | Нет | Количество параллельных задач воркера (по умолчанию `5`) |
| `BULL_STALLED_CHECK_INTERVAL` | Нет | Интервал проверки зависших задач (мс, по умолчанию `30000`) |
| `BULL_LOCK_DURATION` | Нет | Длительность лока задачи (мс, по умолчанию `30000`) |

Полный список параметров с описанием значений по умолчанию приведен в файле [.env.production.example](.env.production.example).
