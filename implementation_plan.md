# План исправления избыточных предупреждений в редакторе изображений

Устранение "шума" в консоли браузера, вызванного предупреждениями о девальвации (deprecation warnings) в сторонней библиотеке `filerobot-image-editor`.

## Причина проблемы
Библиотека `filerobot-image-editor` использует React внутренне. Текущие версии React (18.3+) начали выдавать громкие предупреждения при использовании `defaultProps` в функциональных компонентах. Поскольку библиотека является сторонней и мы используем ее последнюю версию, мы не можем исправить ее исходный код напрямую. Эти предупреждения не влияют на функциональность, но сильно затрудняют чтение логов.

## Шаги реализации

1.  **Анализ `FilerobotEditor.vue`**: Определить точки входа и выхода (lifecycle hooks) для управления состоянием консоли.
2.  **Реализация перехватчика консоли**:
    *   Создать механизм для временной подмены `console.error`.
    *   Добавить логику фильтрации, которая пропускает сообщения, содержащие текст `"Support for defaultProps will be removed from function components"`.
    *   Обеспечить корректное восстановление оригинального `console.error` при уничтожении компонента.
3.  **Обработка нескольких экземпляров**: Использовать счетчик ссылок (reference counting) для корректной работы, если на странице одновременно будет открыто несколько редакторов (хотя в текущем UI это маловероятно).
4.  **Проверка**: Запустить редактор и убедиться, что консоль чиста от данных предупреждений, при этом критические ошибки все еще отображаются.

## Ожидаемый результат
Консоль браузера больше не будет заполнена десятками однотипных предупреждений при открытии редактора изображений, что улучшит опыт разработки и отладки.
