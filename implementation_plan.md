# Исправление ошибки сборки бэкэнда в GitHub Actions

Причина ошибки в том, что тесты Jest пытаются импортировать модули из `@gran/shared`, которые в `package.json` пакета настроены на использование папки `dist`. В GitHub Actions эта папка не создается перед запуском тестов.

## План действий

1. **Обновить GitHub Actions workflow**: Добавить шаг сборки пакета `@gran/shared` перед запуском тестов в файле `.github/workflows/docker-image.yml`.
2. **Настроить Jest для работы с исходниками**: Добавить `moduleNameMapper` в `jest.config.ts`, чтобы Jest искал файлы в `packages/shared/src` напрямую. Это позволит тестам работать даже без предварительной сборки (улучшит локальную разработку).
3. **Проверить работоспособность**: Запустить тесты локально без папки `dist`.
